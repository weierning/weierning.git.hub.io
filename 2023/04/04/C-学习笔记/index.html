C#学习笔记

笔记
发布日期:   2023-04-04
设计模式

单例模式

  • 定义：一个类只能创建一个该类的对象（实列）

  • 实现步骤（以类 A 为例）：
    1、不能创建类的任何一个实例：
    private A( ){ }
    2、只能创建 A 的一个实例（类的静态成员是被类的各个对象所共享的）：
    public static A instance = new A( );
    3、在第一次访问 A 类时创建该实例:
    private static A instance;// = new A;
    public static A Ianstance( )
    {
        if（ instance == null）
       {
          instance = new A( );
        }
    return instance ;

    }

工厂模式
  • 对修改封闭，对增加开放
  • 消灭 switch，改写为虚方法
  • 到底要实例化谁，将来会不会增加实例化对象，考虑用一个单独的类来做这个创造实列的过程
  • 返回别的类型的对象是依赖关系，拥有别的类型的对象是聚合关系，包含别的类似的对象是复合关系。
  • 单一职责原则：就一个类而言，应该只有一个引起它变化的原因。
      三层架构：数据访问层，业务逻辑层，界面显示层
  • 开放封闭原则：面对需求，对程序的改动是增加新代码，而不是改动原有代码。拒绝不成熟的抽象和抽象一样重要。
模板方法模式
  • 当我们要完成某一细节层次一致的一个过程或一系列步骤，但其个别步骤在更详细的层次上实现可能不同时。通常考虑使用模板方法。
  • 实现方法：定义一个操作中的算法的骨架，而将一些步骤延迟到子类中。模板方法使得子类可以不改变一个算法的结构即可重定义该算法某些特定步骤。
接口

  • 规定了从其派生类必须要实现的操作
  • 特别的，一个接口描述了约定的有什么的（What）部分，而类实现了接口描述的如何（How）部分。
  • 接口操作必须是 public 的，所有不需再声明。
  • 一个类只能从一个基类继承，但可以实现多个接口。
  • 如果需要从多个基类继承，可以考虑从一个基类继承，并将其他基类定义为该类的一个成员。
  • 接口（interface）中所有的方法都是 public/abstract 的，不能加 override 关键字，只能包括接口方法
  • 抽象类可从接口继承，再让具向类从抽象类继承。
  • 迪米特法则：如果两个类不彼此直接通信，那么这两个类就不应当发生直接的相互作用。如果其中一个类需要调用另一个类的某一个方法的话，可以通过第三者转发这个调用。
字典
  • 键（key）：字典中的索引
  • 值（value）：字典中某个字的详细解释
  • 键值对的集合就是字典
  • 以电话本为例（用姓名查找电话）：
    internal class Program
    {
       static void Main()
      {
        Dictionary<string,string> phonenumber = new Dictionary<string,string>();
           phonenumber [“张三”]=“023-123456”;
           phonenumber [“李四”]=“023-223456”;
          phonenumber [“王五”]=“023-323456”;
       try
         {
           Console.WriteLine (phonenumber [“赵六”]);
         }
       catch (Exception ex)
          {
           Console.WriteLine(ex.Message);
          }
            Console.ReadLine();
       }
    }


    ref 传参
  • ref 方式传参，实际上是让形参引用实参的地址，形参、实参都要加上 ref
  • 数组在传参的时候，默认传递数组的地址，而值类型的变量（如：int，bool 等）在传参时，默认传递变量的值。


    动态数组
  • ArrayList: 为 object 类型，可以存放任意数组，使用时需要强制转换对象（装箱与封箱）。
  • List<>: 指定为当前数据类型，只能存放当前类似数组，使用时不需转换对象（防止装箱与封箱），也可指定为 object 类型，存放任意数组。
    如：List<“object”> stuList2 = new List <“object”>(); //"" 并不存在。



    构造器

  • 如果一个类没有构造器，C# 会自动为它创建一个不带参数的默认构造器，用户可自行创建构造器取代 C# 的默认构造器。
  • this：调用其他构造器
内部类

  • 概念
     在类中定义的类称为内部类
     内部类外层的类称为外部类
     可以将内部类理解为外部类的一个成员，就像类中的字段、方法成员一样
     内部类默认是 private，可声明为 public
     如果一个内部类是 private 的，那么它就不能作为 public 方法的返回值类型
  • 用途：
     类的层次划分
     将内部类设为私有可禁止外部访问
     内部类可访问外部类的其他同级成员
继承

  • 方法复用代码段的功能
  • 类复用对象的功能
  • 继承，复用基类的功能
  • 语法格式：
     class 派生类：基类
  • 枚举 (enum)：是一个数字，所有枚举值构成一个集合，该集合的枚举值数量是有限的。默认从 0 开始。
     初始化：枚举类型・枚举名称。
  • 当派生类对象被赋值给基类的引用时，此时如果通过基类引用来调用方法，调用的是基类的方法。
     解决方法：基类加 vitrual (虚方法)，派生类加 override，即可对此方法进行重写。
多态

  • 静态多态：
     重载（overload）同名方法：参数多态
     泛型：类型多态
  • 动态多态：
     重写（override）虚方法：行为多态
       实现方法：
         基类的多态方法需要声明为 virtual
         派生类 overrid 基类的同名方法
       在使用时：
         将派生类对象赋值给基类变量
         用基类变量调用多态方法



抽象类


  • 有些类不希望被实例化，可将其创建为抽象类
  • 有些方法只用于规定派生类要实现的功能，而不具体实现某一功能，可以将其创建为抽象方法（abstract），如果一个方法被声明为抽象方法，那么它的类也必须是抽象类。派生类要实现抽象类的方法，需要进行重写。
  • 抽象类不能有任何实现，包括不能有 { }；


    运算符重载

  • 一些运算符可以被重载，使其能够作用于特定的类和结构体，此过程称为运算符重载
  • 一元运算符和除了赋值以外都二元运算符可被重载
      赋值运算符不能被直接重载，但是由于赋值操作是根据算数、逻辑或位移运算产生的结果得来，所以也就间接的得以重载了
  • 重载时机：运算符重载应该仅用于操作数表示的是数字含义的时候，并且能恰当的表达运算符的意思的时候。例如，给一个 Employee 类型的对象 e 提升员工等级时，不应用 e++, 而应该用 e.Promote
  • 如果重载 Equals 方法必须覆盖 Equals 方法
  • 如果覆盖 Equals 方法必须覆盖 GetHanshCode 方法
  • 包含赋值的运算不能被重载
      += 会被拆成 + 和 =
  • 重载转换运算符
      implicit 转换是安全的
      explicit 转换许愿造型
  • 提示：如果转换运算符可抛出异常或返回一个部分结果，应使用 explicit，如果转换可确保工作并且不会造成数据丢失，则可使用 implicit。
  • public static explicit operator Time(int hours)
    {…}// 负数时间不能转换为 Time 对象，所以要求强制转换
  • public static explicit operator Time(Time t1)
    {…}// 转换不会丢失任何内容


    杂记

  • 数组变量存储的不是数组内容，而是数组在内容中的首地址。
  • const 变量的访问方式：类名・const 得到成员名
  • readonly 变量的访问方式：对象名・readonly 的成员名
  • readonly 的赋值有且只有两个地方：
     定义 readonly 变量时直接赋值
     使用构造器
  • 每个对象的 readonly 成员值可以不一样
  • static 对象：一个变量被这个类的所有变量共享。一个类只能有一个静态变量（类名・成员名）
  • debug.Assert (true/false) // 断言（判断一句话是否为真）
